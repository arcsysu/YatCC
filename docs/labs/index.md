# 实验内容总览

YatCC 实验内容按编译器流水线组织，从环境准备、词法分析、语法分析，到中间代码生成、优化与后端代码生成逐步展开。

!!! warning "进入实验内容前先跑通基础链路"

	Task 之间是逐步衔接的。如果环境、构建、评分、打包流程还没有跑通，建议先回到实验指引完成准备，再开始具体实验。
	先把工具链调顺，再做实验，会明显减少无效排错时间。

!!! tip "按流水线顺序推进"

	Task1 到 Task4 都依赖前一阶段的理解或产物。即使某些 Task 提供了复活机制，也更建议你先把上一个阶段的最小正确性打通。
	顺着编译器流水线往前走，定位问题会简单得多。

建议阅读顺序如下：

1. [Task0：环境准备](../task0_doc/task0.md)
2. [Task1：词法分析](../task1_doc/overview.md)
3. [Task2：语法分析](../task2_doc/overview.md)
4. [Task3：中间代码生成](../task3_doc/overview.md)
5. [Task4：中间代码优化](../task4_doc/overview.md)
6. [Task5：后端代码生成](../task5_doc/overview.md)

每个 Task 在顶部页签中的“实验内容”下都保留了一套完整子文档，可以直接通过侧边导航进入对应章节。

!!! note "总览页负责定方向，子页面负责落细节"

	如果你只想快速知道本实验的输入、输出、评分与推荐做法，看总览页就够了。
	如果你已经开始实现，再继续进入对应子页面看 API、框架说明与调试细节。
