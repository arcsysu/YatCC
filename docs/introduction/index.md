# 实验指引

这里汇总了进入 YatCC 编译实验前最常用的准备材料与平台说明。你可以按下面三条路径进入：

!!! note "推荐阅读顺序"

	如果你是第一次进入 YatCC，建议先完成环境配置，再熟悉实验框架和评测流程，最后配置 AI 工具与提示工程。
	这样后续进入 Task1 到 Task5 时，调试、评分和提交链路会顺很多。

!!! warning "不要跳过环境检查直接开始写代码"

	很多看似是实验实现问题的报错，实际来自编译器版本、CMake 缓存或依赖路径配置不一致。
	如果你连最基础的构建目标都还没跑通，先处理环境，再进入具体 Task。

- 环境与流程：环境配置、实验框架使用、评测方式、自动评测平台与配套工具。
- AI 与辅助工具：AI 助教、平台配置、vibe coding 指南与提示工程示例。
- 参与与反馈：文档改进与协作入口。

如果你是第一次进入实验，建议依次阅读以下页面：

1. [环境配置](environment.md)
2. [实验框架使用方法](howtouse.md)
3. [最终评测方式说明](finalscore.md)

如果你需要 AI 辅助完成实验，可以继续阅读：

1. [AI 工具使用方法](agent.md)
2. [vibe coding指南](vibecoding.md)
3. [大模型提示工程](prompt.md)

!!! tip "把这三类资料分开用"

	环境与流程文档解决“怎么跑起来”，实验内容文档解决“这一题要做什么”，AI 相关文档解决“怎样更高效地调试和提问”。
	把用途分清楚，查资料时会快很多。
